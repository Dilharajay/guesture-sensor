# Firmware Fix Report and Codebase Documentation

## 1) Scope
This document covers the `firmware/` codebase (PlatformIO project) and summarizes:
- What was broken
- What was changed to fix it
- How the code works, file-by-file and line-by-line (using line ranges for readability)

---

## 2) Full Fix Report

## 2.1 Baseline command and failures
Command used:
```bash
pio run
```

Initial failures:
1. `src/guesture_inference.cpp:44: fatal error: tensorflow/lite/micro/all_ops_resolver.h: No such file or directory`
2. `src/main.cpp:34: fatal error: eloquent_tinyml/tensorflow.h: No such file or directory`
3. `env:model` failed with: `Error: Nothing to build` because `model_inference.cpp` did not exist but was referenced in `build_src_filter`.

After first fixes, additional failures surfaced:
4. `src/gesture_model.h` parse errors (`expected '}'`, `expected ','`) because byte-array lines were missing commas.
5. Macro collision: Arduino core macro `DEFAULT` conflicted with TFLite profiler enum value `DEFAULT`.
6. CMSIS include failure in EloquentTinyML dependency:
   - `fatal error: cmsis_gcc.h: No such file or directory`

---

## 2.2 Root causes
1. Mixed/inconsistent inference implementations in `src/`:
   - `main.cpp` was intended for EloquentTinyML
   - `guesture_inference.cpp` used direct TFLite Micro include paths that were not present in this PlatformIO dependency setup.
2. Wrong include/API usage in `main.cpp` for the pinned EloquentTinyML version.
3. PlatformIO environment filters referenced a non-existent source file.
4. Generated model header had formatting corruption (missing commas).
5. Arduino macro namespace leakage (`DEFAULT`) broke TFLite headers.
6. Upstream EloquentTinyML package expected `cmsis_gcc.h` but did not ship it in this build path.

---

## 2.3 Files changed and exact fix intent

### A) `platformio.ini`
Changes:
- `env:esp8266`: set `build_src_filter = +<main.cpp>` so the intended inference entrypoint builds.
- Added include path in `build_flags`: `-I include` so local compatibility headers are discoverable.
- Added/fixed `env:model`:
  - `build_src_filter = +<main.cpp>`
  - `lib_deps = ${env:esp8266.lib_deps}`
  - `build_flags = ${env:esp8266.build_flags}`

Why:
- Prevent compilation of incompatible `guesture_inference.cpp` in default inference env.
- Ensure `model` env actually has buildable source and matching dependencies/flags.

### B) `src/main.cpp`
Changes:
- Removed unsupported include path usage and aligned to EloquentTinyML 2.4.0 API.
- Added:
  ```cpp
  #ifdef DEFAULT
  #undef DEFAULT
  #endif
  ```
  before `#include <EloquentTinyML.h>`.
- Updated model wrapper type to:
  `Eloquent::TinyML::TfLite<INPUT_SIZE, N_CLASSES, TENSOR_ARENA_SIZE> ml;`
- Updated load-state/error checks:
  - `isOk()` -> `isInitialized()`
  - `getErrorMessage()` -> `errorMessage()`

Why:
- Match the actual API exposed by this library version.
- Avoid preprocessor macro conflict from Arduino core.

### C) `src/gesture_model.h`
Changes:
- Repaired formatting by restoring trailing commas in the generated byte array initializer.

Why:
- Without commas, the C++ compiler cannot parse the model data array.

### D) `include/cmsis_gcc.h` (new)
Changes:
- Added a compatibility shim providing the missing CMSIS GCC symbols/macros (`__SSAT`, `__USAT`, `__ROR`, `__CLZ`, and common macro definitions).

Why:
- EloquentTinyML/CMSIS chain required this header during kernel compilation.

## 3) Codebase architecture (firmware)

At a high level:
1. **Data collection path** (`src/data_collection.cpp`) logs raw IMU windows over serial.
2. **Inference path** (`src/main.cpp`) runs on-device model inference with EloquentTinyML.
3. **Sensor abstraction** (`src/imu_reader.h`) reads and scales MPU6050 values.
4. **Windowing abstraction** (`src/window_buffer.h`) keeps a sliding 50x6 sample tensor.
5. **Output abstraction** (`src/gesture_output.h`) dispatches recognized gestures over MQTT/WebSocket/MAVLink.
6. **Model blob** (`src/gesture_model.h`) stores quantized TFLite model bytes in flash (`PROGMEM`).
7. **Build orchestration** (`platformio.ini`) selects source files and dependencies per environment.

## 5) End-to-end runtime behavior

Inference runtime (`env:esp8266` or `env:model`):
1. Boot, start serial.
2. Init I2C and MPU6050.
3. Init Wi-Fi + output channels.
4. Load model from `gesture_model.h` into EloquentTinyML.
5. Every 20ms:
   - Read 6-axis IMU sample.
   - Push sample to circular window.
6. Once window has 50 samples:
   - Normalize using training mean/std.
   - Run model inference.
   - Select highest score class.
   - If score >= threshold and cooldown elapsed, dispatch event.
7. Keep MQTT/WebSocket connection loops alive in each pass.

Data collection runtime (`env:data`):
1. Wait for serial command.
2. On `'s'`, begin timed sampling.
3. Output CSV samples for one full window.
4. Emit end marker and stop.

## 7) Security and maintainability recommendations
1. Move Wi-Fi credentials from source file into non-committed config (`build_flags`, secrets header ignored by git, or runtime provisioning).
2. Keep `gesture_model.h` strictly generated from training/export scripts.
3. If `guesture_inference.cpp` is not needed, consider archiving/removing it later to reduce confusion.
4. Consider adding basic unit/integration checks in `test/` for preprocessing consistency and output routing behavior.
